# 人格安全审查清单

> 文本人格仍有红线。法律 / 平台 / 社区三道闸。

## 阻断级（block）

### B1：法律红线

| 类别 | 例 |
|------|---|
| 真实人名 | "Linus Torvalds 风格人格" |
| 商标 / 角色 IP | "钢铁侠工程师"（迪士尼）、"绝地武士"（卢卡斯影业） |
| 名人语录直引 | 长段引用且无授权 |
| 商业产品代言式人设 | "<某产品> 官方助手" |

→ 必须改写或匿名化。

### B2：平台审查

| 类别 | 例 |
|------|---|
| 政治立场 | "支持 X 党 / 反对 Y 政府" |
| 宗教推崇 | "信仰 X 教，传播 Y 经典" |
| 民族 / 地域歧视 | "<地域> 人都怎样" |
| 性别 / 取向贬低 | 任何对群体的负面刻画 |

→ 全平台分发会被拦，必拒。

### B3：内容危害

| 类别 | 例 |
|------|---|
| 鼓励真实自残 / 自杀 | 任何形式 |
| 鼓励违法（非 CTF / 非授权研究） | 实际意义上的恶意指导 |
| 性化未成年角色 | 任何形式 |
| 仇恨言论 | 任何群体 |

→ 一票否决。

### B4：技术红线

| 检查 | 阻断条件 |
|------|---------|
| voice.self | 单字符、符号、空字符串 |
| voice.user | 同上 |
| identity.md | 缺角色锚定 / 性格特征 / 情绪模式任一段 |
| scenarios | 数组为空 |
| capabilities.authorization | 安全相关 persona 但未声明 tier |

## 警告级（warn）

| 检查 | 触发 |
|------|------|
| 与内置人格差异度 < 30% | 建议改进现有而非新建 |
| description 营销腔 | "powerful"、"the best"、"comprehensive" |
| emoji_policy vs register 矛盾 | formal + heavy / casual + none |
| identity.md > 4000 字 | 过长，建议精简 |
| identity.md 写成 prompt | "You are..."、"You should..."—应叙述式 |

## 信息级（info）

- 缺 tags → 影响发现
- 缺 author 联络方式 → 维护者无法回询
- 未提供 ≥ 3 个 scenarios → 覆盖面薄弱

## 自检话术

向用户提议沉淀 / 提交时，附以下自检 prompt：

```
请确认你的人格不含：
  □ 真实人名 / 商标 / IP
  □ 政治 / 宗教 / 民族立场
  □ 鼓励违法 / 自残 / 仇恨内容
  □ 性化未成年角色

确认后吾继续校验技术维度。
```

魔尊确认后再跑机器校验。

## 反模式

- ❌ 跳过自检直接发 PR → 维护者必拒
- ❌ "我只是开玩笑"作为豁免理由 → 不豁免
- ❌ 引用现有作品但不署名 → 加来源说明或改写
