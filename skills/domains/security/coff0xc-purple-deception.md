# Coff0xc · 紫队与欺骗防御

## 触发

purple team、ATT&CK、红蓝对抗、control validation、detection coverage、emulation plan、honeypot、deception、decoy、canary、检测有效性、蓝队发现攻击行为、覆盖指标、改进闭环、防守能力验证。

## 边界

- 演练必须有授权、范围、时间窗、停止条件和业务影响控制。
- 不输出可用于未授权攻击的完整操作链。
- 蜜罐/欺骗部署需确认隔离、法律、隐私和告警噪声。

## 能力矩阵

| 能力域 | 覆盖 | 要点 |
| --- | --- | --- |
| ATT&CK 映射 | 技术、战术、数据源、检测逻辑 | 行为到遥测 |
| 演练计划 | 场景、范围、节奏、角色、停止条件 | ROE |
| 检测验证 | 规则命中、字段完整性、延迟、误报 | 可量化 |
| 响应验证 | triage、升级、遏制、恢复、复盘 | runbook |
| 欺骗防御 | honeytoken、canary、decoy service、假凭证 | 隔离和监控 |
| 指标 | coverage、MTTD、MTTR、FP rate、closure rate | 闭环 |

## 工作流

1. 目标定义：验证哪些控制、哪些 ATT&CK 技术、哪些业务资产。
2. 场景设计：分步行为、预期日志、检测点、停止条件。
3. 基线确认：现有规则、日志源、EDR/SIEM 字段。
4. 演练执行：授权环境内记录时间线和观测结果。
5. 差距分析：未命中原因、字段缺失、误报、响应断点。
6. 改进闭环：规则、日志、流程、培训、复测计划。

## 验证清单

- 每个演练动作映射 ATT&CK 和数据源。
- 检测覆盖有正/负样例。
- 告警质量有指标和 owner。
- 欺骗资产隔离且有退场方案。
- 复测确认改进有效。

## 反模式

- 演练只证明“能打”，不验证“能防”。
- 没有日志字段和查询证据。
- 蜜罐产生大量噪声无人处理。
- 没有把复盘转成 backlog。
