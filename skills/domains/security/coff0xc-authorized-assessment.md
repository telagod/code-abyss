# Coff0xc · 授权评估与控制验证

## 触发

recon、fingerprint、attack chain、full pentest、red team、phishing simulation、post-exploitation、data exfiltration 演练、CDN/WAF、ROE、书面授权、完整入侵链、防护发现、边界和步骤、防御验证。

## 边界

- 没有书面授权和范围时，只输出评估计划模板、授权清单和防御建议。
- 不提供真实第三方目标的定制利用链、凭证获取、持久化、规避检测或 C2 操作。
- 钓鱼、外传、社工、后渗透等演练必须有 ROE、联系人、停止条件和数据处理规则。

## 能力矩阵

| 能力域 | 覆盖 | 要点 |
| --- | --- | --- |
| ROE/授权 | 资产、时间窗、禁止动作、速率、联系人、停止条件 | 先确认边界 |
| 攻击面梳理 | 域名、应用、API、云资产、供应商、身份入口 | 防御视角 |
| 控制验证 | WAF、EDR、SIEM、IAM、备份、日志、DLP | 可测指标 |
| 红队防御化 | attack path、检测点、遏制点、复盘 | 不输出滥用细节 |
| 钓鱼演练 | 培训目标、模板审批、落点隔离、统计 | 合规和隐私 |
| 外传演练 | 合成数据、DLP 验证、速率/大小限制 | 不用真实敏感数据 |
| 报告 | 范围、证据、影响、控制失效、修复路线 | 可执行 |

## 工作流

1. 授权确认：书面范围、资产、时间窗、联系人、停止条件。
2. 目标建模：业务目标、关键资产、已知控制、日志覆盖。
3. 场景设计：攻击链拆成可控步骤，每步有检测和回滚。
4. 执行记录：只在授权环境中操作，记录时间、工具、命令、证据。
5. 防御映射：每个行为映射 ATT&CK、控制点、日志源、告警情况。
6. 复盘闭环：差距、优先级、owner、SLA、复测方案。

## 验证清单

- ROE 已确认并可引用。
- 每个测试动作都有停止条件。
- 使用合成数据，不触碰真实敏感数据。
- 每个发现有检测/缓解建议。
- 报告区分攻击路径和防御改进。

## 反模式

- 没授权就进入操作细节。
- 把红队计划写成不可控攻击手册。
- 没有日志和控制验证指标。
- 演练结束没有复测和改进 owner。
