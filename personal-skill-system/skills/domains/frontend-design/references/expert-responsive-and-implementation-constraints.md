# Expert Responsive And Implementation Constraints

## Use

Use this when a design must survive small screens, slow devices, or limited engineering budget.

## Rules

- responsive design changes priority, not only width
- do not ship a visual system that collapses under content growth
- choose design complexity that the implementation can preserve consistently
- accessibility and performance regressions count as design failures

## Output

- responsive priority changes
- implementation constraints
- performance risks
- fallback behavior
