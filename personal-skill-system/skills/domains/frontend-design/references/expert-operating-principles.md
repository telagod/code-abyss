# Expert Operating Principles

Use this reference when the task needs top-tier product and interface judgement rather than
generic layout cleanup.

## Order of judgement

1. user goal
2. information hierarchy
3. interaction clarity
4. accessibility and readability
5. visual language
6. performance and implementation cost

## Core rules

- hierarchy beats decoration
- distinctive visual language should still preserve usability
- accessibility is a baseline constraint, not a polish pass
- motion should explain state, not distract from it
- component systems should create coherence without flattening product character

## Design questions

- what must users notice first
- what action or decision should feel easiest
- where can the interface become ambiguous or noisy
- what happens on narrow screens or slow devices
- what makes this design memorable rather than generic

## Output contract

Produce:

- hierarchy map
- interaction stance
- accessibility considerations
- visual direction
- responsive behavior
- implementation risk notes
