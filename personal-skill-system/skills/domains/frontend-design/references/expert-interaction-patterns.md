# Expert Interaction Patterns

## Use

Use this when the main failure is unclear actions, weak feedback, or state transitions that feel arbitrary.

## Rules

- every action needs visible affordance, feedback, and failure handling
- primary actions should not compete with secondary actions
- destructive actions need friction proportional to blast radius
- transitions should preserve continuity of user intent

## Output

- primary actions
- feedback model
- error or empty-state behavior
- transition risks
