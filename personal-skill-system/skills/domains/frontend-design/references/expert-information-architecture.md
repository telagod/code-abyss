# Expert Information Architecture

## Use

Use this when the hard part is deciding what users should notice, in what order, and with what navigation cost.

## Rules

- one screen should make one primary decision easiest
- group by task, not by implementation source
- navigation depth is a tax; charge it only when information density justifies it
- labels should reduce ambiguity, not mirror internal naming

## Output

- primary user goal
- hierarchy order
- navigation structure
- confusion risks
