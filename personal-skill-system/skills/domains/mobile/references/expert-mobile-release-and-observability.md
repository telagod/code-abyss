# Expert Mobile Release And Observability

## Use

Use this when crash visibility, staged rollout, client-side health, or mobile release safety matter.

## Rules

- app-store or phased release changes rollback speed
- client logs and crash signal must answer what broke on device, not only on server
- feature flags should support safe disablement after release
- version skew is a permanent constraint

## Output

- rollout strategy
- client observability plan
- rollback or kill-switch notes
