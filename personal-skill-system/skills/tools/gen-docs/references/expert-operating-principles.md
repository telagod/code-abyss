# Expert Operating Principles

Use this reference when documentation generation must create a durable engineering scaffold rather
than empty ceremony.

## Core rules

- document boundaries before implementation trivia
- make failure modes and operational assumptions visible early
- generated docs should accelerate real filling, not create dead templates
- if a module has external contracts, mention them explicitly

## Strong scaffold outputs

A strong scaffold should make room for:

- purpose
- ownership
- boundary and inputs
- outputs and side effects
- dependencies
- failure modes
- verification path
